## 1. bitXor

```c
//这个是使用位运算的操作来实现符号^(异或操作)
//异或就可以写成 (A&非B)&(非A&B)
//要求是只能使用~,&
//这个简单,或操作可以用与非代替
//所以就直接写了
int bitXor(int x,int y){
    return ~(~(~x&y)&~(x&~y));
}
```

## 3.isTMax

```c
//是否是32位有符号数的最大值
//是返回1,不是返回0
//目前不会写,2023/11/25

//2023/11/28，学习
int isTMax(int x){
  //第一次思考
  //首先一个数比如x,那么x^x就是0
  //所以明显tmax就是01111 1111(先用八位表示)
  //也就是说假设这个数是tmax的话，我们应该通过某种手段来得到一个数y,使得y=x,那么再将y与x异或
  //观察一下,让tmax+1后就是tmin,那么tmin按位取反之后就是tmax
  //所以如果x是tmax的话,那么一定满足x=~(x+1),x^(~(x+1))=0
  //也就是如果是tmax的话,要求我们返回1,可以用逻辑否定!
  //但是有一个数比较特殊
  //那就是1111 1111,我们发现这个数加1之后是0,按位取反后还是1111 1111,显然这个数也满足我们的假设,所以得把这个数排除掉
  //也就是 ((x+1)^0x0)得是0才可以,那么就是(!!(x+1)^0x0),因为这是算数与(&),所以我们得两次取反,而很明显根据数学知识,两次取反相当于负负得正,所以只是把非0的数转换为1而不会影响我们前面的那个数的结果(1&1=1,1&0=0,前面那个结果不是1就是0)
    return (!(~(x+1)^x))&((~(x+1)^0x0));
}
```

## 4.allOddBits

```c
/* 
	*奇数位上的数字全是1,比如0XAAAAAAA就是1010 1010,第一位是从0开始计数，最后一位是第31位,所以这个数返回1
 *   Rating: 2
 */
//目前不会做,2023/11/25
//其实就是看这个数是不是0xAAAAAAAA就是,偶数位可以通过&来消除
int allOddBits(int x) {
  //我们知道0xA就是1010
  //那么我们想要得到0xAA怎么办呢,那么就是把0xA左移四位即可,就是0x1010 0000,那么低四位就是0000,根据运算符或,我们就可以把这低四位补上即可
  int A=0xA;
  int AA=A<<4|A;//此时A是4位,AA就是8位
  int AAAA=AA<<8|AA;//位移8位的意思就是2的幂次方,因为低八位就已经是0xAA了,0xAAAA就是16位,此时再向前移动16位,最后得到0xAAAAAAAA;
  int num=AAAA|(AAAA<<16);//num是32位
  //再来思考一下,如果一个数x是0xAAAAAAAA,那么x&0xAAAAAAAA的结果就是0xAAAAAAAA(num),那么我们就可以通过异或操作来判断一下这个数x是不是,单反这个数奇数位上有一位不是1,那么num&x的结果肯定不是num了,
  return !((num&x)^num)
}

```
